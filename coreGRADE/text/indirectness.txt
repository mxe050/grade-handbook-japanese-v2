GRADEアプローチにおける「非直接性 (Indirectness)」の実践的解説

ご提示いただいた図は、エビデンス（論文などの研究結果）を目の前の患者や目的とする状況に当てはめる際の「ズレ」、すなわち**「非直接性（Indirectness）」**について説明しています。

元の図が示している通り、非直接性は大きく以下の2つのカテゴリーに分類されます。

間接比較 (Indirect comparisons)：図の左側。「比較の経路」に関する非直接性。

PICOの非直接性 (PICO indirectness)：図の右側（赤枠）。「臨床疑問と論文データのズレ」に関する非直接性（★GRADEアプローチにおける最大の焦点）。

それぞれの要素について、実践的な観点から詳細に解説します。

① 間接比較 (Indirect comparisons) とネットワークメタ分析

図の左側に示されているのは、主に**「ネットワークメタ分析（Network Meta-Analysis: NMA）」**という手法を用いた際に発生する非直接性です。この領域は専門用語が多く混乱しやすいため、図の「A」「B」「C」の関係に沿って用語を丁寧に整理します。

【図の状況解説】

図では、新薬である治療「A」と治療「B」のどちらが優れているか（A vs B）を知りたいと考えています。しかし、現実の論文データは以下のようになっていました。

C と A の間（太い実線）：「A vs C」を直接比較したランダム化比較試験（RCT）が3つある。

C と B の間（太い実線）：「B vs C」を直接比較したRCTが8つある。

A と B の間（点線）：「A vs B」を直接比較した研究が存在しない。

この状況で、共通する「C」を介して、数学的に「AとB」を比較する手法が用いられます。

【ネットワークメタ分析の用語整理】

現場で混乱しやすい類義語も含めて、図の要素を解説します。

直接比較 (Direct comparison)

意味: 2つの治療法（例：AとC）を、同じ一つの臨床試験の中で直接戦わせて比較すること。

別名: Head-to-Head（ヘッド・トゥ・ヘッド）比較、直接エビデンス。

図での表現: 太い青色の実線。最も信頼性が高い比較方法です。

間接比較 (Indirect comparison)

意味: 直接比較した試験がない2つの治療法（AとB）について、共通の対照（C）を介して間接的に効果の差を計算・推測すること。

別名: 間接的エビデンス。

図での表現: 青色の点線。「AとCの差」と「BとCの差」を引き算して「AとBの差」を導き出します。

共通対照 (Common comparator)

意味: 間接比較の「橋渡し」となる治療法のこと。

図での表現: 円「C」。過去の標準治療薬やプラセボ（偽薬）がこのCの役割を果たすことが非常に多いです。

ネットワークメタ分析 (Network meta-analysis / NMA)

意味: 上記の「直接比較」と「間接比較」のデータをすべて統合し、3つ以上の治療法（A, B, C...）の有効性や安全性を同時に比較し、ランキング付けなどを行う高度な統計手法のこと。

【実践的な注意点：なぜ間接比較は確実性が下がるのか？】

間接比較（A vs B）が正しく成立するためには、「A vs C の試験に参加した患者」と「B vs C の試験に参加した患者」の背景（重症度、年齢、併存疾患など）が、非常に似通っていることが絶対条件となります（これを「推移性の仮定」と呼びます）。

しかし現実には、異なる時期や場所で行われた別々の試験の患者背景が完全に一致することは稀です。そのため、「Cという別の試験結果をフィルターとして通している分、直接比較よりも『ズレ（バイアス）』が入り込んでいる可能性が高い」とみなされます。
これが「間接比較による非直接性」の正体であり、GRADEアプローチにおいて間接比較のエビデンスは、**原則として直接比較のエビデンスよりも確実性を1段階下げる（ダウングレードする）**ことになっています。

② PICOの非直接性 (PICO indirectness) 【重要】

図の右側、赤枠で囲まれている部分です。通常のガイドライン作成において「非直接性」と言えば、主にこちらを指します。「Target PICO（私たちが知りたい疑問）」に対して、「Search results（実際に見つかった論文）」がどれだけズレているかを表しています。

概念図解（的と矢のメタファー）

図では的（まと）と矢を使って、この概念を見事に表現しています。

🎯 的の中心（Target PICO）：あなたが知りたい、理想的でドンピシャの臨床疑問。

📄 検索結果（Search results）：文献検索で実際に見つかった論文。

🎯🏹 直接（Direct）：見つかった論文が、的のど真ん中に刺さっている状態（知りたいことと論文の条件が完全に一致）。

🎯↘️ 間接（Indirect）：見つかった論文が、的の端の方に刺さっている状態（少しズレているが、参考にはなる）。

【実践的な解説：PICOのズレとは？】

PICOとは、臨床疑問を構成する4要素（Patient, Intervention, Comparison, Outcome）です。論文を目の前の患者に適用する際、以下の4つのズレをチェックします。

1. P (Patient/Population：患者) のズレ

Target：重症の新型コロナ患者に対する治療効果を知りたい。

見つかった論文：軽症〜中等症の患者を対象とした論文しかなかった。

実践判断：軽症者に効くからといって、重症者にも効くとは限りません。小児のデータがないため成人のデータから推測する場合なども非直接性となります。

2. I (Intervention：介入) のズレ

Target：薬剤Xを「1日3回」投与した場合の効果を知りたい。

見つかった論文：薬剤Xを「1日1回」投与した論文しかなかった。あるいは、同系統の少し古い「薬剤W」の論文しかなかった。

実践判断：投与量や投与方法が違ったり、同クラスの別薬のデータを用いたりする場合、効果や副作用が完全に同じとは言えません。

3. C (Comparison：比較対照) のズレ

Target：新薬Aと「現在の標準治療薬B」を比較したい。

見つかった論文：新薬Aと「プラセボ（偽薬）」を比較した論文しかなかった。

実践判断：プラセボより優れていることは分かっても、今の標準治療から乗り換えるべきかは分かりません。臨床現場の疑問との間に大きなズレが生じます。

4. O (Outcome：アウトカム) のズレ（※現場で最も頻出！）

Target：その薬を飲むことで「死亡率が下がるか」「心筋梗塞を防げるか」（真のアウトカム）を知りたい。

見つかった論文：「血圧が下がった」「コレステロール値が改善した」（代用アウトカム）というデータしかなかった。

実践判断：検査値が良くなっても、患者の寿命が延びたり生活の質が上がったりするとは限りません。代用アウトカムを見たら「非直接性あり」と強く疑うべきです。

現場でのアクション：どう評価するか？

これらの「ズレ（非直接性）」を見つけた場合、エビデンスの確実性（Certainty of evidence）を**ダウングレード（格下げ）**します。

ズレがない、または非常に小さい（Direct）：ダウングレードしない。

少しズレがあるが推測可能（Indirect）：確実性を「1段階下げる」。

ズレが大きすぎて参考にしづらい（Very Indirect）：確実性を「2段階下げる」。

まとめ

ご提示いただいた画像は、「理想のデータ」と「現実の論文データ」の間には必ずギャップがあり、**①比較経路のズレ（間接比較）**と、**②PICO条件のズレ（PICOの非直接性）**という2つの視点から、そのギャップを批判的に吟味しなければならないという、EBMの実践において極めて重要な概念を簡潔に表現した名図解です。

間接性の定義と範囲

非常に包括的。PICOのズレによる間接性に加え、直接の比較（Head-to-head）がないことによる「間接比較（ネットワークメタアナリシスなど）」も同じIndirectnessのドメイン内で広く議論される。

「間接比較（ネットワークメタアナリシス等）」と「PICO要素の不一致による間接性」を明確に区別。Core GRADEでは単一介入・単一対照の「PICO要素のズレ」に焦点を絞って評価する。

評価のプロセス

概念的・定性的。対象集団、介入、対照、アウトカムの違いが、効果の推定値に系統的かつ意味のある違いをもたらすと考えられる場合にダウングレードするという一般的な指針。

**アルゴリズム化（構造化）**されている。PICOの各要素について「違いの有無」→「効果への影響の類似性（生物学的・社会的要因）」へとステップバイステップで判断を下す。

ダウングレードの基準

深刻な間接性（-1）か、非常に深刻な間接性（-2）かを、評価者の主観的・総合的な判断に委ねる部分が大きい。

判断ツリー（フローチャート）により、どの程度似ていないか（Dissimilar / Very dissimilar）や、サロゲートアウトカムの因果経路の距離（Close / Far）に基づいて、機械的かつ透明性をもって -1 または -2 の判断ができるように設計されている。

サロゲート指標の扱い

サロゲート（代用）アウトカムを用いた場合は間接性としてダウングレードの対象となる、という原則的な記載。

ターゲットとする真のアウトカム（患者にとって重要なエンドポイント）に至る**因果経路上の距離（Very close / Close / Far）**に応じて、ダウングレードしない〜2段階下げる、という具体的な基準を提示。

2. Core GRADEにおける非直接性の評価手順（詳細版）

添付のLetter to Editor（JCE 2026）およびフローチャート（図 5-2）に基づく、具体的な非直接性の評価手順です。

このプロセスは、**「関心のある臨床的疑問（Target PICO）」と、「実際に利用可能な研究エビデンスのPICO（Study PICO）」**を比較することから始まります。

ステップ1：P / I / C（集団・介入・対照）の差異の特定

まず、対象集団（P）、介入（I）、対照（C）のそれぞれについて、ターゲットと実際の研究との間に違いがあるかを確認します。

Population（対象集団）: 年齢層、時代の変化による対象者の違い、状態や疾患の重症度などに違いはないか？

Intervention（介入）: 介入の強度/用量、投与期間、投与経路、術者のスキル（習熟度）に違いはないか？ あるいは、ノンアドヒアランス（服薬不遵守）、治療の切り替え、技術の変遷などの問題はないか？

Comparator（対照）: 対照群の治療の強度/用量、期間、投与経路、管理の質に違いはないか？

【判定】

「No (違いはない)」の場合: ダウングレードしません（Do not rate down）。

「Yes (違いがある)」の場合: ステップ2へ進みます。

ステップ2：P / I / C の生物学的・社会的要因による影響の評価

ステップ1で「違いがある」と判断された場合、単に違うからといってすぐにダウングレードするわけではありません。以下の問いに答えます。

問い： 「研究対象の患者/介入/対照とターゲットとの間に違いはあるが、生物学的または社会的な要因は十分に類似しており、ターゲットとなるアウトカムに対して同様の影響を与えると予想できるか？」

【判定】

「Yes (影響は同じと予想できる)」: ダウングレードしません（Do not rate down）。

「No (Dissimilar: 似ていない)」: 確実性を 1段階ダウングレード（Rate down once） します。

「No (Very dissimilar: 全く似ていない)」: 確実性を 2段階ダウングレード（Rate down twice） します。

ステップ3：Outcome（アウトカム）の評価

アウトカムについては、P/I/Cとは別の評価ルートをたどります。まず、設定されたアウトカムが「サロゲート（代用）アウトカム」であるかどうかを確認します。

ルートA：サロゲートアウトカムである場合（Yes）

測定されたアウトカムが、本当に知りたい患者にとって重要なエンドポイントの「代用品（例：死亡率の代わりに血圧、骨折の代わりに骨密度など）」である場合、以下の問いに進みます。

問い： 「そのサロゲートアウトカムは、患者にとって重要なエンドポイントに至る推定上の因果経路の中で、どのくらい離れているか？」

【判定】

Very close（非常に近い）: ダウングレードしません（Do not rate down）。

※例外規定（図の注釈より）：サロゲートとみなされるものであっても、疾患プロセスに極めて密接かつ直接的に結びついており、事実上患者にとって重要なアウトカムを常に表すものがあります（例：レントゲンで診断された股関節骨折は必然的に痛みや治療の必要性をもたらす。マラリアにおけるPCR検査での治療失敗確認など）。これらは間接性で落としません。

Close（近い）: 確実性を 1段階ダウングレード（Rate down once） します。

Far（遠い）: 確実性を 2段階ダウングレード（Rate down twice） します。

ルートB：サロゲートアウトカムではない場合（No）

直接的なアウトカムであっても、測定方法や期間のズレがないかを評価します。

問い： 「研究で用いられたアウトカムを定義する基準や、**測定のタイムフレーム（期間）**は、ターゲットとしているものと類似しているか？」

【判定】

「Yes (類似している)」: ダウングレードしません（Do not rate down）。

「No (類似していない)」: 確実性を 1段階ダウングレード（Rate down once） します。

まとめ

Core GRADEのアルゴリズムは、PICOのズレを発見した際に、「本当に効果推定値に影響を与えるズレなのか（生物学的・社会的要因）」「因果経路としてどの程度離れているのか」を段階的に問うことで、評価者ごとのブレをなくし、より透明性の高いダウングレード（0段階、-1段階、-2段階）の判断ができるように構築されています。
